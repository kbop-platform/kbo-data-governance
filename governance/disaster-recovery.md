# 재해 복구 기본 정책

> 최종수정: 2026-02-25 | 거버넌스 정책
> 📋 문서 성격: 재해 복구 정책(안)

## 1. 목적

KBO 데이터 시스템의 장애 및 재해 상황에서 데이터 복구 절차와 목표를 정의한다.

## 2. 복구 목표

| 구분 | RTO (복구 시간 목표) | RPO (복구 시점 목표) | 비고 |
|------|---------------------|---------------------|------|
| **시즌 중 (3~10월)** | 4시간 | 1시간 | 경기 당일은 최우선 대응 |
| **비시즌 (11~2월)** | 24시간 | 24시간 | 익영업일 내 복구 |
| **포스트시즌** | 2시간 | 30분 | 방송 대응 필수 |

## 3. 백업 정책

| 대상 | 주기 | 보존 기간 | 방식 |
|------|------|----------|------|
| DB1 (영구 데이터) | 일 1회 | 90일 (롤링) + 연말 영구 | 전체 백업 |
| DB2 (시즌 데이터) | 일 1회 (시즌 중) | 시즌 종료 후 1년 | 전체 백업 |
| DB2 경기 당일 | 경기 종료 후 즉시 | 30일 | 차등 백업 |
| 마스터 데이터 | 변경 시 즉시 | 영구 | 스냅샷 |

## 4. 복구 절차

### 4.1 데이터 손상/유실 시

1. 장애 인지 및 범위 파악
2. 최근 백업에서 복원
3. S2i에 누락 데이터 재전송 요청 (경기 데이터의 경우)
4. 데이터 정합성 검증 (FK 무결성, PK 중복, 행 수 대조)
5. 복구 완료 보고 및 원인 분석

### 4.2 시스템 장애 시

1. 장애 인지 (모니터링 알림 또는 수동 확인)
2. 장애 원인 분류 (DB 서버 / 네트워크 / S2i 전송)
3. 임시 조치 (읽기 전용 전환, 캐시 서빙 등)
4. 근본 원인 해결 및 서비스 복구
5. 사후 보고서 작성

## 5. 담당자 연락 체계

| 순위 | 역할 | 연락 방법 | 응답 목표 |
|------|------|----------|----------|
| 1차 | 데이터 관리자 (R-01) | 내부 메신저 | 30분 내 |
| 2차 | 기록위원 (R-03) | 전화 | 1시간 내 |
| 3차 | S2i 운영 담당 | 계약 핫라인 | 2시간 내 |

> 구체적 연락처는 별도 내부 문서로 관리 (보안 사유)

## 6. 연간 점검

| 항목 | 주기 | 담당 |
|------|------|------|
| 백업 복원 테스트 | 반기 1회 | 데이터 관리자 |
| 복구 절차 모의 훈련 | 연 1회 | 전체 팀 |
| RTO/RPO 적정성 검토 | 연 1회 | 데이터 관리자 |
