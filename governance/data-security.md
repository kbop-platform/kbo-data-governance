# 데이터 보안 및 프라이버시 정책

> 최종수정: 2026-02-25 | 거버넌스 정책

## 1. 목적

KBO 데이터 자산의 보안 등급을 정의하고, 접근 권한과 개인정보 처리 방침을 수립한다.

## 2. 데이터 분류 체계

| 등급 | 정의 | 예시 | 접근 범위 |
|------|------|------|----------|
| **Public** | 공개 가능 데이터 | 팀 코드, 구장 코드, 경기 일정, 경기 결과 | 전체 (외부 API 포함) |
| **Internal** | 내부 업무용 데이터 | 경기 기록 상세, 투수·타자 기록, 통계 | KBO 내부 + 계약 파트너 |
| **Restricted** | 개인정보·민감 데이터 | 선수 생년월일, 연봉, 주소, 연락처 | 인가된 담당자만 |

## 3. 테이블별 보안 등급

| 도메인 | 테이블 | 등급 | 사유 |
|--------|--------|------|------|
| master | person, person2, PERSON | Restricted | 선수 PII (생년월일, 신체 정보) |
| master | PERSON_FA | Restricted | FA 계약 정보 |
| master | TEAM | Public | 공개 팀 정보 |
| master | STADIUM | Public | 공개 구장 정보 |
| master | KBO_schedule | Public | 공개 일정 |
| master | CANCEL_GAME | Public | 공개 취소 정보 |
| game | GAMEINFO | Internal | 심판·기록원 이름 포함 |
| game | Hitter, Pitcher, Score 등 | Internal | 비공개 상세 기록 |
| stats | BatTotal, PitTotal 등 | Internal | 가공 통계 |
| realtime | IE_* | Internal | 실시간 운영 데이터 |

## 4. 접근 권한 원칙

### 4.1 최소 권한 원칙 (Principle of Least Privilege)
- 각 역할은 업무 수행에 필요한 최소한의 데이터만 접근
- 기본값: 접근 불가 → 필요 시 권한 부여

### 4.2 역할 기반 접근 제어 (RBAC)

| 역할 | Public | Internal | Restricted |
|------|--------|----------|------------|
| 데이터 관리자 (R-01) | ✅ | ✅ | ✅ |
| 기록위원 (R-03) | ✅ | ✅ | ✅ (선수 정보) |
| 통계 분석가 (R-04) | ✅ | ✅ | ❌ |
| 외부 API 사용자 | ✅ | ❌ | ❌ |
| 수행사 개발자 | ✅ | ✅ (개발 환경) | ❌ |

→ 참고: [데이터 오너십](data-ownership.md) — 역할 정의 (R-01~R-06)

### 4.3 접근 권한 변경 절차
1. 권한 요청서 제출 (요청자 → 데이터 관리자)
2. 데이터 오너 승인
3. 시스템 반영 (1영업일 내)
4. 분기별 접근 권한 검토

## 5. 개인정보(PII) 처리 방침

### 5.1 PII 항목 정의

| 항목 | 테이블.컬럼 | 처리 방침 |
|------|------------|----------|
| 선수 이름 | person.NAME | 공개 (공인) |
| 생년월일 | person.BIRTH | Restricted (연도만 공개 가능) |
| 신장/체중 | person.HEIGHT, WEIGHT | Internal (미디어 카드용) |
| 연봉 | PERSON_FA 관련 | Restricted |
| 등록/말소일 | person.REG_DT 등 | Internal |

### 5.2 비식별화 원칙
- 외부 제공 시 PII 마스킹 또는 제거
- 통계 목적: 집계값만 제공 (개인 식별 불가 수준)
- 연구 목적: 비식별화 데이터셋 별도 생성

### 5.3 보존 및 파기
- 선수 기록: 영구 보존 (야구 역사 기록 가치)
- PII: 선수 은퇴 후 5년간 유지, 이후 비식별화 처리
- 로그 데이터: 3년 보존 후 파기

## 6. 보안 점검

| 항목 | 주기 | 담당 |
|------|------|------|
| 접근 권한 검토 | 분기 1회 | 데이터 관리자 |
| 보안 등급 재분류 | 연 1회 | 데이터 관리자 |
| PII 처리 현황 점검 | 연 1회 | 데이터 관리자 |
| 외부 접근 로그 검토 | 월 1회 | 데이터 관리자 |
